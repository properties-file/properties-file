# Migrating to v2

> This guide covers migrating from v1 to v2. Each migration guide is designed to upgrade from the immediately preceding major version.

## Overview

Version 2 is a complete rewrite of the original package created by [@calibr](https://github.com/calibr). The v1 package provided basic `parse`/`stringify` functions. Version 2 replaces these with a fully Java-spec-compliant implementation that supports Unicode escapes, multiline values, comments, and more.

### What changed

| v1                                               | v2                                                        |
| ------------------------------------------------ | --------------------------------------------------------- |
| `parse(string)` returns `{ key: value }`         | `propertiesToJson(filePath)` or `getProperties(filePath)` |
| `stringify(object)` returns `.properties` string | Removed (use `escapeKey`/`escapeValue` to build manually) |
| Basic `=` splitting, no escape handling          | Full Java `Properties` spec compliance                    |
| Silent on malformed input                        | Throws on malformed Unicode escapes (like Java)           |
| No comment or multiline support                  | Full support for comments, multiline, all separator types |
| Single `index.js` file                           | TypeScript with ESM/CJS support                           |

## Migration steps

### Replacing `parse()`

_v1_

```js
var parser = require('properties-file')
var object = parser.parse(propertiesFileContents)
console.log(object)
```

_v2 — simple key-value object (closest equivalent)_

```js
const { propertiesToJson } = require('properties-file/content')
const object = propertiesToJson(propertiesFileContents)
console.log(object)
```

_v2 — from a file path_

```js
const { propertiesToJson } = require('properties-file')
const object = propertiesToJson('path/to/file.properties')
console.log(object)
```

### Replacing `stringify()`

There is no direct replacement for `stringify()` in v2. If you need to convert an object back to `.properties` format, you can use the escape utilities:

_v1_

```js
var parser = require('properties-file')
var string = parser.stringify({ key1: 'value1', key2: 'value2' })
```

_v2_

```js
const { getProperties } = require('properties-file/content')
// Build .properties content manually using the escape utilities.
const lines = Object.entries(myObject).map(([key, value]) => `${key} = ${value}`)
const string = lines.join('\n')
```

### Behavior differences

- **Unicode escapes**: v1 did not process `\uXXXX` sequences. In v2, `\u00E9` is decoded to `é`. Malformed sequences like `\uhello` will throw an error.
- **Multiline values**: v1 treated `\` at end of line as literal. In v2, it means line continuation (matching Java behavior).
- **Comments**: v1 included comment lines as keys. In v2, lines starting with `#` or `!` (after leading whitespace) are properly skipped.
- **Separators**: v1 only split on `=`. In v2, `=`, `:`, and whitespace are all valid separators (matching Java behavior).
- **Leading whitespace**: v1 preserved leading whitespace in keys. In v2, leading whitespace is stripped (matching Java behavior).
