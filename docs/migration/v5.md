# Migrating to v5

> This guide covers migrating from v4 to v5. Each migration guide is designed to upgrade from the immediately preceding major version.

## Overview

Version 5 is a major release that replaces the `Properties` class with a **lossless data model**. Every element in a `.properties` file — properties, comments, blank lines, leading whitespace, duplicate keys — is now preserved and accessible. The `PropertiesEditor` has been rewritten to work on this new model.

### What changed

| v4                                                                | v5                                                                                                |
| ----------------------------------------------------------------- | ------------------------------------------------------------------------------------------------- |
| `import { Properties } from 'properties-file'`                    | `import { Properties } from 'properties-file/parser'`                                             |
| `import { PropertiesEditor } from 'properties-file/editor'`       | `import { PropertiesEditor } from 'properties-file/editor'` (same path, new API)                  |
| `properties.collection` (`Property[]`, deduplicated)              | `properties.nodes` (`PropertiesNode[]`, all elements in order)                                    |
| `properties.getKeyCollisions()` returns `{ startingLineNumbers }` | `properties.getKeyCollisions()` returns `{ nodes: PropertyNode[] }`                               |
| `PropertiesEditor.delete(key, false)` returns `boolean`           | `PropertiesEditor.delete(key, { deleteLeadingNodes: false })` returns `PropertyNode \| undefined` |
| `PropertiesEditor.deleteAll(key)` not available                   | `PropertiesEditor.deleteAll(key)` returns `PropertyNode[]`                                        |
| Comments and blank lines discarded during parsing                 | Comments and blank lines preserved as `CommentNode` and `BlankLineNode`                           |
| `Property` class with linked-list pointers                        | `PropertyNode` plain object                                                                       |
| `PropertyLine` class                                              | Removed (logic absorbed into parser)                                                              |

### What stayed the same

- `import { getProperties } from 'properties-file'` — unchanged API, same behavior
- `import { escapeKey, escapeValue } from 'properties-file/escape'` — unchanged
- `import { unescapeContent } from 'properties-file/unescape'` — unchanged
- All bundler integrations (Webpack, Rollup/Vite, esbuild, Bun) — unchanged
- Full Java `Properties` specification compliance — unchanged

## Migration steps

### 1. Update `Properties` import path

The `Properties` class has moved from the main entry to the `/parser` sub-path. The main entry now only exports `getProperties`.

_v4_

```ts
import { Properties } from 'properties-file'
```

_v5_

```ts
import { Properties } from 'properties-file/parser'
```

### 2. Replace `properties.collection` with `properties.nodes` or query methods

The `collection` property has been replaced by `nodes`, which contains **all** elements (properties, comments, blank lines) in file order, including duplicate keys.

_v4_

```ts
properties.collection.forEach((property) => {
  console.log(`${property.key} = ${property.value}`)
})
```

_v5 — iterating only properties_

```ts
properties.getProperties().forEach((node) => {
  console.log(`${node.key} = ${node.value}`)
})
```

_v5 — iterating all nodes_

```ts
import { PropertiesNodeType } from 'properties-file/parser'

for (const node of properties.nodes) {
  switch (node.type) {
    case PropertiesNodeType.PROPERTY:
      console.log(`${node.key} = ${node.value}`)
      break
    case PropertiesNodeType.COMMENT:
      console.log(`${node.delimiter} ${node.body}`)
      break
    case PropertiesNodeType.BLANK:
      console.log('(blank line)')
      break
  }
}
```

### 3. Update `Property` references to `PropertyNode`

The `Property` class has been replaced by the `PropertyNode` type (a plain readonly object).

_v4_

```ts
import { Property } from 'properties-file/property' // No longer exists
```

_v5_

```ts
import type { PropertyNode } from 'properties-file/parser'
```

Key differences in the property data:

| v4 `Property`                                         | v5 `PropertyNode`                                                                  |
| ----------------------------------------------------- | ---------------------------------------------------------------------------------- |
| `property.separator` (full string like `" = "`)       | `node.separatorLeading`, `node.separatorChar`, `node.separatorTrailing` (granular) |
| `property.linesContent`                               | `node.rawLines` (array of physical lines)                                          |
| `property.previousProperty` / `property.nextProperty` | Not needed — use array position in `properties.nodes`                              |
| `property.hasKeyCollisions`                           | Use `properties.getKeyCollisions()`                                                |

### 4. Update `getKeyCollisions()` usage

The return type now includes the actual property nodes instead of just line numbers.

_v4_

```ts
const collisions = properties.getKeyCollisions()
collisions.forEach((collision) => {
  console.log(`Key '${collision.key}' on lines ${collision.startingLineNumbers.join(', ')}`)
})
```

_v5_

```ts
const collisions = properties.getKeyCollisions()
collisions.forEach((collision) => {
  const lines = collision.nodes.map((node) => node.startingLineNumber)
  console.log(`Key '${collision.key}' on lines ${lines.join(', ')}`)
  // You now also have access to each node's value, separator, etc.
})
```

### 5. Update `PropertiesEditor` import and API

The `PropertiesEditor` is still at `properties-file/editor` but extends the new `Properties` class.

_v4 — delete without removing comments (returns boolean)_

```ts
properties.delete('key', false)
```

_v5 — returns the deleted PropertyNode (or undefined if not found)_

```ts
const deleted = properties.delete('key', { deleteLeadingNodes: false })
if (deleted) {
  console.log(`Deleted key '${deleted.key}' with value '${deleted.value}'`)
}
```

_v4 — update with no options_

```ts
properties.update('key') // allowed, did nothing
```

_v5_

```ts
properties.update('key', {}) // options object is required
```

### 6. New capabilities in v5

These features were not available in v4:

```ts
import { Properties } from 'properties-file/parser'

const properties = new Properties(content)

// Lossless round-trip: format() reproduces the exact original content.
properties.format() === content // true

// Access comments and blank lines.
properties.getComments() // CommentNode[]
properties.getBlankLines() // BlankLineNode[]

// Get all occurrences of a duplicate key.
properties.getPropertyNodes('key') // PropertyNode[] (all occurrences)

// Get comment/blank nodes before a property.
properties.getLeadingNodes('key') // (CommentNode | BlankLineNode)[]

// Normalize output with granular control.
properties.format({
  removeComments: true,
  removeBlankLines: true,
  deduplicateKeys: true,
  separatorChar: '=',
  separatorLeading: ' ',
  separatorTrailing: ' ',
  escapeUnicode: true,
  collapseMultiline: true,
})
```

```ts
import { PropertiesEditor } from 'properties-file/editor'

const editor = new PropertiesEditor(content)

// Delete ALL occurrences of a duplicate key (new in v5).
editor.deleteAll('key') // returns PropertyNode[] of deleted nodes
```
